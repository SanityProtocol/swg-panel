# swg-panel

Self-hosted панель управления небольшим сервисом WireGuard / AmneziaWG на одном или нескольких входных серверах. Панель — единственный источник истины о пирах; каждый входной сервер (**узел**) синхронизируется с ней по **исходящему HTTPS** и приводит свой набор пиров к назначенному — поэтому узлам не нужен входящий доступ, SSH или rsync.

Рассчитано на небольшие доверенные развёртывания — десятки пользователей, несколько серверов — с прицелом на многозвенную схему (входной узел на краю → удалённый выход).

## Как это работает

- **Панель** (управляющий слой) — отдаёт веб-интерфейс, собственный TLS + логин и API синхронизации узлов. Хранит реестр (ваши пиры) и список узлов. Чистый Python, без базы данных.
- **Узел** (входной сервер) — запускает `swg-noded`, который каждые несколько секунд отправляет снимок своего интерфейса панели, получает набор пиров, которые на нём *должны* быть, и приводит интерфейс к этому состоянию через `swg-agent` (добавляя/удаляя пиры на «живом» wg/awg-интерфейсе). Только исходящий HTTPS.
- **Декларативно** — вы меняете пиры в панели; узлы подтягиваются сами. Узел, пропустивший такт, восстановится при следующей синхронизации, а кратковременный сбой панели никогда не сотрёт пиры узла (узел применяет изменения только при корректном ответе).
- **Ключи остаются у владельца** — пары ключей пиров генерируются в вашем браузере. Приватный ключ попадает в конфиг/QR, который показывается **один раз**, и никогда не передаётся на сервер; панель хранит лишь публичный ключ, выданный IP и общий ключ (PSK), чтобы держать все узлы согласованными.

## Method × Role

Два независимых выбора.

**Method** — как устанавливается:
- **bare-metal** — службы systemd, через `install-host.sh` / `install-node.sh`.
- **docker** — `docker compose` с образами `swg-panel` / `swg-node`.

**Role** — что делает машина:
- **host** — только панель.
- **host+node** — панель *и* эта машина также входной сервер (подключается автоматически).
- **node** — только входной сервер.

Можно смешивать: Docker-панель с bare-metal-узлами, bare-metal `host+node` плюс дополнительные Docker-узлы и т. д.

## Компоненты

| компонент | где | назначение |
|---|---|---|
| `swg-panel-server` | хост | интерфейс + API синхронизации + реестр/список узлов; отдаёт собственный TLS + логин и панель статуса |
| `swg-noded` | каждый узел | отправляет снимки, получает нужный набор пиров, согласует локально — только исходящий HTTPS |
| `swg-agent` | каждый узел | применяет одно добавление/удаление пира на «живом» интерфейсе и в его `.conf` |
| `users.json` | хост | **реестр** — ваши пиры (публичный ключ, IP, PSK и на каких узлах живёт каждый) |
| `nodes.json` | хост | **список узлов** — имена, адреса и хеши токенов подключения |

Вручную `users.json` и `nodes.json` не редактируются — это делает интерфейс.

## Быстрый старт

### A — bare-metal

Запустите одну команду на каждой машине; роль — **host** (только панель) или **host+node** (панель, которая ещё и входной сервер) — выбирается во время установки:

```
curl -fsSL https://raw.githubusercontent.com/SanityProtocol/swg-panel/main/bootstrap.sh | sudo bash -s host
```

Установщик проведёт через роль, TLS и логин администратора и сам настроит интерфейс wg/awg: покажет найденные (выберите, какими управлять), предложит **создать** один, если их нет, и затем спросит, нужно ли **добавить ещё** — каждый новый интерфейс может быть **AmneziaWG** (с профилем обфускации 2.0, который использует этот проект) или обычным **WireGuard**, и автоматически получает NAT в интернет. После установки откройте `https://<ваш-хост>/`, войдите и:

- Для **host+node** эта машина уже в разделе **Nodes** — добавляйте пиры.
- Для **host** добавьте каждый входной сервер из интерфейса: **Nodes → Add node** напечатает одноразовый токен и точную команду `install-node.sh` (см. [Добавление узла](#добавление-узла)).

### B — Docker

```
git clone https://github.com/SanityProtocol/swg-panel && cd swg-panel
cp .env.example .env        # задайте PANEL_PASSWORD (и PANEL_DOMAIN)
docker compose --profile host up -d
```

Откройте `https://<хост>:8443/`. Про узел или панель-с-локальным-узлом см. [Docker](#docker).

## Установка панели

`install-host.sh` (запускается через `bootstrap.sh host` или напрямую из клона) ставит панель и, для `host+node`, локальный входной сервер. Запрашивает:

| вопрос | смысл |
|---|---|
| **Role** | `host` (только панель) или `host+node` (ещё и входной сервер) |
| **Имя узла / публичный IP** | *(только host+node)* идентификатор и адрес, который набирают клиенты |
| **Интерфейсы** | *(только host+node)* какими существующими интерфейсами wg/awg управлять; предложит **создать**, если их нет, затем **добавить ещё** — каждый AmneziaWG (обфускация 2.0) или обычный WireGuard. Новые интерфейсы получают серверный ключ, подсеть и автоматический forward + masquerade (`PostUp`/`PostDown`). |
| **Домен / IP панели** | имя хоста или IP, по которому доступна панель |
| **Порт** | по умолчанию **443** (unit добавит право на привязку для портов < 1024) |
| **Режим TLS** | `selfsigned` · `letsencrypt` · `cloudflare` · `manual` · `none` |
| **Режим отдачи** | `standalone` (по умолчанию, самодостаточный) или `nginx` (за обратным прокси) |
| **Пользователь / пароль администратора** | веб-логин — предлагает `admin` + 3 цифры; пароль вводится и подтверждается (меняется позже в разделе **Account**) |

**Режимы TLS:**
- **selfsigned** — мгновенно; браузер один раз предупредит. Подходит для старта или за туннелем.
- **letsencrypt** — настоящий сертификат через `acme.sh --standalone`; **нужен свободный порт 80** для выпуска *и* продления.
- **cloudflare** — настоящий сертификат через DNS-01; **порт 80 не используется**. Токену нужны права `Zone:DNS:Edit` + `Zone:Read`.
- **manual** — вы предоставляете `fullchain.pem` + `key.pem`.
- **none** — обычный HTTP (логин идёт в открытом виде — только за туннелем).

Пример автоматической установки (конфиг через переменные окружения):

```
sudo -E ROLE=host TLS_MODE=cloudflare CF_TOKEN=… PANEL_DOMAIN=panel.example.net \
     BASIC_USER=admin BASIC_PASS='…' bash -s host \
     < <(curl -fsSL https://raw.githubusercontent.com/SanityProtocol/swg-panel/main/bootstrap.sh)
```

## Добавление узла

Узлы управляются целиком из интерфейса — установщик больше о них не спрашивает.

1. **Nodes → Add node** — задайте имя, публичный IP/хост, который будут набирать клиенты, и цвет. Вы получите **одноразовый токен подключения** и точную команду для запуска.
2. **На входном сервере**, где уже настроен wg/awg, выполните эту команду (скрипт берётся из клона репозитория):
   ```
   sudo PANEL_URL="https://panel.example.net" NODE_TOKEN="<token>" ./install-node.sh
   ```
   Он управляет интерфейсом wg/awg так же, как установщик панели: показывает существующие, предлагает **создать** (AmneziaWG 2.0 или обычный WireGuard), если их нет, спрашивает про **добавить ещё**, сам настраивает forward + masquerade, затем ставит `swg-noded` и направляет его на панель.
3. Через несколько секунд узел станет **online** в разделе Nodes.

Если панель использует самоподписанный сертификат, узел по умолчанию его не проверяет (учётные данные — это токен, а канал всё равно зашифрован). Чтобы проверять, ответьте «да» на вопрос о TLS при настоящем сертификате или закрепите самоподписанный, передав `TLS_FINGERPRINT=<sha256-hex>` (его sha256 проверяется во время TLS-рукопожатия, ещё до отправки токена).

**Действия над узлом** в интерфейсе: **Edit** (адрес/цвет — заданный здесь адрес попадает в конфиги клиентов), **Rotate token** (старый токен немедленно перестаёт работать; узел нужно подключить заново) и **Remove** (отзывает токен и снимает узел со всех пиров).

## Управление пирами

**Пир** — это одна личность (пара ключей + IP + PSK), которую можно развернуть сразу на нескольких узлах для резервирования: клиент переключается между ними, меняя `Endpoint`.

- **Add peer** — выберите интерфейс и один или несколько узлов, при желании имя. Панель выделяет свободный IP; браузер генерирует пару ключей и PSK и собирает конфиг(и). Конфиг + QR показываются **один раз** — приватный ключ не сохраняется.
- **Назначение / снятие** — развёртывание той же личности на новых узлах просто добавляет туда её публичный ключ и PSK; в конфиге клиента появляется ещё один endpoint для переключения.
- **Копировать на другой сервер** — со страницы пира скопируйте его на сервер, где его ещё нет. Используются тот же ключ и PSK, но выдаётся **новый IP** из подсети того сервера (отдельный туннель с общей личностью). Нужен приватный ключ клиента, поэтому работает при включённом `store_configs` или сразу после создания пира.
- **Remove** — убирает пир из реестра; каждый узел, где он был, удалит его при следующей синхронизации.

Статус (online, partial, dangling, …) вычисляется при каждом обновлении по снимкам узлов — пир остаётся «online», пока одна из реплик ненадолго недоступна.

**Account** — имя пользователя и пароль панели меняются в разделе **Account**; изменение вступает в силу сразу (вас попросят войти заново).

## Docker

Один compose-файл, три профиля:

```
docker compose --profile host up -d         # только панель
docker compose --profile node up -d         # только входной сервер
docker compose --profile host-node up -d    # панель + локальный входной сервер
```

Настройка через `.env` (скопируйте из `.env.example`):

- **Панель:** `PANEL_PASSWORD` (обязательно), `PANEL_USER`, `PANEL_DOMAIN`, `TLS` (`selfsigned`|`none`), `PANEL_PORT`.
- **Узел:** `PANEL_URL` (для `host-node` используйте `https://swg-panel:8443`), `NODE_TOKEN` (из раздела Nodes), `NODE_ENDPOINT`, `NODE_IFACE`, `NODE_LISTEN_PORT`, `NODE_ADDRESS`, `TLS_VERIFY`, `DNS`.

Порядок тот же, что и на bare-metal: поднимите панель, **Nodes → Add node** для получения токена, задайте `NODE_TOKEN`, затем запустите профиль узла.

**Два образа.** `swg-panel` — чистый Python (небольшой, низкий риск). `swg-node` собирает userspace-датаплейн **`amneziawg-go`** и инструменты `awg` — контейнер не может загрузить модуль ядра хоста — и требует `NET_ADMIN` + `/dev/net/tun`. Без смонтированного конфига по умолчанию генерируется интерфейс **AmneziaWG 2.0** (H-диапазоны, S1–S4, консервативный QUIC `I1`); задайте `NODE_PLAIN_WG=yes` для обычного WireGuard или смонтируйте свой `.conf` в `/etc/swg-node/<iface>.conf`. Masquerade для подсети туннеля добавляется автоматически (compose уже задаёт `ip_forward` + `NET_ADMIN`). Для максимальной пропускной способности предпочтительнее **bare-metal**-узел с модулем ядра.

## Справочник по конфигурации

**Панель — `/etc/swg-panel/fleet.json`** (см. `fleet.example.json`):

```json
{
  "roster_path":   "/var/lib/swg-panel/users.json",
  "nodes_path":    "/var/lib/swg-panel/nodes.json",
  "stats_dir":     "/var/www/wgstats",
  "store_configs": false,
  "config_dir":    "/var/lib/swg-panel/configs",
  "node_interval": 5
}
```

`store_configs` хранит сгенерированные конфиги клиентов (включая приватные ключи) на панели — по умолчанию выключено; не включайте без понимания последствий.

**Узел — `/etc/swg-agent/config.json`** (см. `config.example.json`):

```json
{
  "interfaces": { "awg0": { "cmd": ["awg"], "conf": "/etc/amnezia/amneziawg/awg0.conf" } },
  "endpoint_host": "203.0.113.7",
  "dns": ["1.1.1.1"],
  "panel": { "url": "https://panel.example.net", "token": "…", "verify": false },
  "node":  { "interval": 5, "agent": "/opt/swg-agent/swg-agent", "sudo": false }
}
```

Узел может обслуживать несколько интерфейсов — перечислите их все в `interfaces`; каждый пир применяется к тому, которому принадлежит. Поставьте `panel.verify: true` для настоящего сертификата или добавьте `panel.fingerprint: "<sha256-hex>"`, чтобы закрепить самоподписанный.

## Эксплуатация

- **Логи:** `journalctl -u swg-panel-server -f` (панель), `journalctl -u swg-noded -f` (узел). Docker: `docker compose logs -f`.
- **Сменить токен узла:** Nodes → ⋯ → Rotate token, затем повторите команду установки (или обновите `config.json` узла).
- **Удалить узел:** Nodes → Remove. Чтобы вывести машину из строя, остановите на ней `swg-noded`.
- **Резервная копия:** `users.json` + `nodes.json` (в `/var/lib/swg-panel`) — это всё состояние. Скопируйте их в надёжное место.
- **Удаление:** `… | sudo bash -s uninstall` (или `./uninstall.sh`). Останавливает службы и удаляет файлы панели/узла; спрашивает перед тем, как трогать wg/awg, и может сохранить реестр и список узлов для переустановки.

## Безопасность

- **Транспорт:** узлы всегда подключаются только *наружу* к панели по TLS. Предпочтителен настоящий сертификат (`letsencrypt`/`cloudflare`), чтобы узлы работали с `verify: true`; с самоподписанным — закрепляйте отпечаток.
- **Токены узлов** аутентифицируют узел перед панелью. Показываются один раз, хранятся только в виде хеша, могут быть сменены. Считайте `config.json` узла (где лежит «живой» токен) секретом — он в режиме `600`.
- **PSK** генерируются для каждого пира и хранятся в реестре, чтобы все узлы оставались согласованными; держите `/var/lib/swg-panel` доступным на чтение только пользователю панели.
- **Приватные ключи** генерируются в браузере и не передаются на сервер (если вы намеренно не включили `store_configs`). Если ключ пира утерян — выпустите пир заново.

## Диагностика

- **Узел остаётся «awaiting enroll» / не подключается.** Он ещё не синхронизировался. Смотрите `journalctl -u swg-noded -f` на узле: `fingerprint mismatch` — неверный отпечаток; HTTP `401` — неверный или сменённый токен; ошибка соединения — не тот URL/порт панели или закрыт фаервол.
- **Узел «offline».** Он синхронизировался раньше, но замолчал — проверьте демон и исходящую сеть узла.
- **Пиры не появляются на узле.** Убедитесь, что пир назначен на этот узел и его интерфейс совпадает с тем, что узел действительно обслуживает. Данные берутся из последнего снимка узла, поэтому только что добавленному узлу нужна одна синхронизация.
- **Браузер предупреждает о сертификате.** Это нормально при `selfsigned`. Для продакшена используйте настоящий сертификат или примите предупреждение за доверенным туннелем.
- **Клиент подключается (рукопожатие проходит), но нет интернета / `rx` растёт, а `tx` ≈ 0.** Узел расшифровывает пакеты, но не маршрутизирует их наружу. Интерфейсы, созданные установщиком, настраивают это автоматически; для интерфейса, сделанного вручную, включите форвардинг (`net.ipv4.ip_forward=1`) и добавьте masquerade для подсети туннеля через WAN-интерфейс (`iptables -t nat -A POSTROUTING -s <subnet> -o <wan> -j MASQUERADE`).
- **`acme.sh` не смог выпустить сертификат.** Для `letsencrypt` нужен доступный порт 80; если 80 занят — используйте `cloudflare` (DNS-01, без порта 80).
